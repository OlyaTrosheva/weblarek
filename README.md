# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Компоненты работы с данными (Часть 1)

### Раздел: Данные (Интерфейсы)

В приложении используются следующие основные интерфейсы для описания структуры данных:

* **IProduct** — описывает структуру данных одного товара в магазине.
  * `id: string` — уникальный идентификатор товара.
  * `title: string` — название товара.
  * `image: string` — ссылка на картинку товара.
  * `category: string` — категория товара (например, «софт», «хард»).
  * `price: number | null` — цена товара (может быть null, если товар недоступен).
  * `description: string` — полное описание товара.

* **IBuyer** — описывает данные покупателя из формы оформления заказа.
  * `payment: TPayment` — выбранный способ оплаты (`'card' | 'cash' | ''`).
  * `address: string` — адрес доставки.
  * `email: string` — электронная почта покупателя.
  * `phone: string` — номер телефона покупателя.

* **IOrderPost** — описывает финальный объект заказа, отправляемый на сервер бэкенда.
  * `total: number` — итоговая сумма заказа.
  * `items: string[]` — массив уникальных ID купленных товаров.

### Раздел: Модели данных (Классы)

Для управления состоянием приложения и хранения данных реализованы три независимых класса-модели:

#### 1. Класс `CatalogModel`
Отвечает за хранение списка товаров, доступных для покупки, и управление выбранной для превью карточкой.
* **Конструктор:** Параметров не принимает. При инициализации создает пустые хранилища для товаров.
* **Поля класса:**
  * `_items: IProduct[]` — приватный массив всех загруженных товаров.
  * `_preview: IProduct | null` — товар, выбранный пользователем для подробного отображения в модальном окне.
* **Методы класса:**
  * `setItems(items: IProduct[]): void` — сохраняет массив товаров, полученный из внешних источников (API).
  * `getItems(): IProduct[]` — возвращает массив всех товаров из модели.
  * `getProduct(id: string): IProduct | undefined` — находит и возвращает один товар по его уникальному ID.
  * `setPreview(item: IProduct | null): void` — сохраняет товар для подробного отображения.
  * `getPreview(): IProduct | null` — возвращает товар, выбранный для подробного отображения.

#### 2. Класс `BasketModel`
Отвечает за хранение товаров, которые пользователь выбрал для покупки, и логику корзины.
* **Конструктор:** Параметров не имеет. Инициализирует пустую корзину.
* **Поля класса:**
  * `_items: IProduct[]` — приватный массив товаров, добавленных в корзину.
* **Методы класса:**
  * `getItems(): IProduct[]` — возвращает массив товаров, которые сейчас находятся в корзине.
  * `add(product: IProduct): void` — добавляет товар в массив корзины (с проверкой на дубликаты).
  * `remove(id: string): void` — удаляет товар из массива корзины по его ID.
  * `clear(): void` — полностью очищает корзину.
  * `getTotal(): number` — вычисляет и возвращает суммарную стоимость всех товаров в корзине.
  * `getCount(): number` — возвращает количество товаров в корзине (для счетчика на главной).
  * `hasProduct(id: string): boolean` — проверяет наличие товара в корзине по его ID.

#### 3. Класс `BuyerModel`
Отвечает за хранение данных покупателя, точечное изменение полей и их валидацию. Реализует интерфейс `IBuyer`.
* **Конструктор:** Параметров не имеет. Сбрасывает поля в начальные пустые значения.
* **Поля класса:**
  * `payment: TPayment` — вид оплаты.
  * `address: string` — адрес доставки.
  * `phone: string` — телефон покупателя.
  * `email: string` — email покупателя.
* **Методы класса:**
  * `setField(field: keyof IBuyer, value: string): void` — сохраняет данные в конкретное поле модели, не затирая данные в других полях.
  * `getBuyerData(): IBuyer` — возвращает объект со всеми текущими данными покупателя.
  * `clear(): void` — полностью очищает данные покупателя.
  * `validate(): TBuyerErrors` — выполняет валидацию данных формы. Возвращает объект с текстом ошибок для каждого невалидного (пустого) поля. Если ошибок нет — возвращает пустой объект.

### Раздел: Слой коммуникации

#### Класс `LarekApi`
Представитель коммуникационного слоя. Отвечает за получение данных с бэкенда и отправку заказов. Использует композицию, принимая экземпляр базового класса `Api` для выполнения низкоуровневых сетевых запросов.

* **Конструктор:**
  * `constructor(api: Api)` — принимает объект базового класса `Api`, соответствующий интерфейсу `IApi`.
* **Методы класса:**
  * `getProductsList(): Promise<IProductsResponse>` — делает GET-запрос на эндпоинт `/product/` и возвращает объект с сервера, содержащий массив товаров.
  * `postOrder(order: IOrderPost): Promise<IOrderResult>` — делает POST-запрос на эндпоинт `/order/`, передавая в него данные заказа, и возвращает объект, подтверждающий покупку на определенную сумму.


## Слой представления (View)

Все компоненты слоя представления наследуются от базового класса `Component<T>` и отвечают за визуальное отображение данных, установку слушателей событий на элементы интерфейса и передачу сигналов в Презентер.

### 1. Страница приложения (`Page`)
Класс управляет элементами главной страницы: каталогом товаров, счетчиком корзины и блокировкой прокрутки.
*   **Конструктор**: `constructor(container: HTMLElement, events: IEvents)` — принимает корневой контейнер страницы и брокер событий.
*   **Поля**:
    *   `_counter: HTMLElement` — элемент счетчика товаров в шапке.
    *   `_catalog: HTMLElement` — контейнер-галерея для карточек товаров.
    *   `_wrapper: HTMLElement` — главный селектор обертки страницы.
    *   `_basket: HTMLElement` — элемент кнопки корзины в шапке.
*   **Методы (Сеттеры)**:
    *   `set counter(value: number)` — устанавливает текстовое значение счетчика.
    *   `set catalog(items: HTMLElement[])` — заменяет содержимое галереи массивом готовых карточек.
    *   `set locked(value: boolean)` — включает/выключает CSS-модификатор `page__wrapper_locked` для блокировки скролла.
*   **Метод `render`**: `render(data?: Partial<IPage>): HTMLElement` — принимает частичные данные страницы и возвращает её DOM-элемент.

### 2. Карточка товара (`Card`)
Универсальный компонент для отображения карточки товара в каталоге, модальном окне предпросмотра или списке корзины.
*   **Конструктор**: `constructor(container: HTMLElement, actions?: ICardActions)` — принимает шаблон карточки и объект с обработчиками событий (клик).
*   **Поля**:
    *   `_title: HTMLElement` — название товара.
    *   `_image?: HTMLImageElement` — изображение товара (опционально).
    *   `_category?: HTMLElement` — плашка категории товара (опционально).
    *   `_price: HTMLElement` — элемент отображения цены.
    *   `_text?: HTMLElement` — описание товара (опционально).
    *   `_button?: HTMLButtonElement` — кнопка действия (опционально).
    *   `_index?: HTMLElement` — порядковый номер товара в корзине (опционально).
*   **Методы (Сеттеры)**:
    *   `set id(value: string)` — записывает ID товара в `data-id` контейнера.
    *   `set title(value: string)` — устанавливает текст заголовка.
    *   `set image(value: string)` — устанавливает ссылку на картинку и `alt`.
    *   `set category(value: string)` — подставляет текст категории и динамически меняет CSS-класс на основе объекта `categoryMap`.
    *   `set price(value: number | null)` — выводит цену или текст «Бесценно» (при `null` блокирует кнопку текстом «Недоступно»).
    *   `set text(value: string)` — устанавливает текст описания.
    *   `set buttonText(value: string)` — меняет текст на кнопке действия.
    *   `set index(value: number)` — устанавливает индекс товара для корзины.
*   **Метод `render`**: `render(data?: Partial<ICard>): HTMLElement` — наполняет карточку данными и возвращает её.

### 3. Модальное окно (`Modal`)
Компонент-обертка для управления открытием и закрытием модальных окон.
*   **Конструктор**: `constructor(container: HTMLElement, events: IEvents)` — находит элементы управления и вешает события закрытия по крестику и оверлею.
*   **Поля**:
    *   `_closeButton: HTMLButtonElement` — кнопка-крестик закрытия окна.
    *   `_content: HTMLElement` — контейнер для динамического содержимого модалки.
    *   `events: IEvents` — брокер событий.
*   **Методы**:
    *   `set content(value: HTMLElement)` — заменяет внутренний контент модального окна новым элементом.
    *   `open(): void` — открывает модалку (добавляет класс `modal_active`) и инициирует событие `modal:open`.
    *   `close(): void` — закрывает модалку, очищает контент и инициирует событие `modal:close`.
    *   `render(data?: Partial<IModalData>): HTMLElement` — принимает контент, вставляет его, вызывает `open()` и возвращает контейнер.

### 4. Компонент корзины (`Basket`)
Отвечает за отображение списка выбранных товаров и их суммарной стоимости.
*   **Конструктор**: `constructor(container: HTMLElement, events: IEvents)` — находит список, поле цены и вешает на кнопку «Оформить» вызов события `order:open`.
*   **Поля**:
    *   `_list: HTMLElement` — контейнер для списка товаров.
    *   `_total: HTMLElement` — элемент для вывода общей стоимости.
    *   `_button: HTMLButtonElement` — кнопка оформления заказа.
*   **Методы (Сеттеры)**:
    *   `set items(items: HTMLElement[])` — рендерит массив карточек товаров в корзине. Если массив пуст — выводит текст «Корзина пуста» и блокирует кнопку оформления.
    *   `set total(total: number)` — выводит итоговую стоимость в синапсах.
*   **Метод `render`**: `render(data?: Partial<IBasketView>): HTMLElement` — возвращает заполненный элемент корзины.

### 5. Базовый компонент формы (`Form<T>`)
Родительский класс для всех форм в приложении. Управляет инпутами, валидацией и сабмитом.
*   **Конструктор**: `constructor(protected container: HTMLFormElement, protected events: IEvents)` — вешает слушатели событий `input` на все поля и событие `submit` на форму.
*   **Поля**:
    *   `_submit: HTMLButtonElement` — кнопка отправки формы.
    *   `_errors: HTMLElement` — контейнер для вывода текста ошибок валидации.
*   **Методы**:
    *   `protected onInputChange(field: keyof T, value: string): void` — генерирует событие изменения конкретного поля ввода для Презентера.
    *   `set valid(value: boolean)` — управляет доступностью кнопки отправки (`disabled`).
    *   `set errors(value: string | string[])` — выводит текст ошибок под формой.
    *   `render(data?: Partial<IFormState> & T): HTMLElement` — обновляет состояние валидации, ошибок, заполняет инпуты и возвращает форму.

### 6. Форма заказа (`Order`)
Наследник класса `Form`. Отвечает за выбор способа оплаты и адрес доставки на первом шаге.
*   **Конструктор**: `constructor(container: HTMLFormElement, events: IEvents)` — инициализирует кнопки выбора оплаты и вешает на них события.
*   **Поля**:
    *   `_cardButton: HTMLButtonElement` — кнопка выбора оплаты онлайн.
    *   `_cashButton: HTMLButtonElement` — кнопка выбора оплаты при получении.
*   **Методы (Сеттеры)**:
    *   `set payment(value: TPayment)` — переключает класс активности `button_alt-active` на кнопках оплаты.
    *   `set address(value: string)` — устанавливает значение в инпут адреса.

### 7. Форма контактов (`Contacts`)
Наследник класса `Form`. Отвечает за ввод email и телефона на втором шаге заказа.
*   **Конструктор**: `constructor(container: HTMLFormElement, events: IEvents)` — вызывает конструктор родителя.
*   **Методы (Сеттеры)**:
    *   `set email(value: string)` — устанавливает значение в инпут почты.
    *   `set phone(value: string)` — устанавливает значение в инпут телефона.

### 8. Окно успеха (`Success`)
Компонент для отображения финального экрана успешной покупки.
*   **Конструктор**: `constructor(container: HTMLElement, actions: ISuccessActions)` — находит элементы и привязывает клик по кнопке «За новыми покупками!» к закрытию окна.
*   **Поля**:
    *   `_close: HTMLButtonElement` — кнопка закрытия окна.
    *   `_description: HTMLElement` — текст с описанием количества списанных синапсов.
*   **Методы (Сеттеры)**:
    *   `set total(value: number)` — выводит итоговую сумму списания.
*   **Метод `render`**: `render(data?: Partial<ISuccess>): HTMLElement` — возвращает DOM-элемент окна успеха.

## Событийная модель (Связующий слой Presenter)

Взаимодействие слоев организовано через брокер событий `EventEmitter` по паттерну MVP. Презентер (код в файле `main.ts`) выступает в роли диспетчера: он только слушает события (`on`), запрашивает данные из моделей и напрямую вызывает рендер интерфейса, не генерируя события самостоятельно.

### Список ключевых событий в приложении:

#### События Моделей данных (Model):
*   `items:changed` — массив товаров каталога успешно загружен с сервера или изменен.
*   `preview:changed` — выбран товар для детального просмотра в модальном окне.
*   `basket:changed` — изменился состав товаров в корзине (добавление или удаление).
*   `buyer:changed` — изменились данные покупателя, передается объект с результатами валидации.

#### События Элементов представления (View):
*   `card:select` — пользователь кликнул по карточке в каталоге для открытия превью.
*   `basket:open` — нажата иконка корзины в шапке сайта.
*   `order:open` — нажата кнопка «Оформить» внутри корзины.
*   `order:submit` — заполнен первый шаг заказа, нажата кнопка «Далее».
*   `contacts:submit` — заполнен второй шаг заказа, нажата кнопка «Оплатить».
*   `order.<поле>:change` — пользователь вводит данные в поля формы первого шага.
*   `contacts.<поле>:change` — пользователь вводит данные в поля формы второго шага.
*   `modal:open` — модальное окно открыто (необходимо заблокировать прокрутку страницы).
*   `modal:close` — модальное окно закрыто (необходимо разблокировать прокрутку страницы).
